# BrainFuck Compiler

Учебный проект — компилятор языка BrainFuck на Python.  
Принимает `.bf` файл, строит AST-дерево, проверяет ошибки, оптимизирует и выполняет программу через встроенную виртуальную машину.

---

## Запуск

```bash
python main.py <файл.bf>
```

### Флаги

| Флаг | Описание |
|------|----------|
| `--no-run` | Только показать AST-дерево, не выполнять программу |
| `--no-opt` | Отключить оптимизатор |

### Примеры

```bash
python main.py hello.bf
python main.py hello.bf --no-run
python main.py hello.bf --no-opt
```

---

## Язык BrainFuck

BrainFuck — минималистичный язык программирования. Работает с лентой из 30 000 ячеек и указателем на текущую ячейку. Всё остальное в исходном файле считается комментарием и игнорируется.

| Команда | Значение |
|---------|----------|
| `+` | Увеличить текущую ячейку на 1 |
| `-` | Уменьшить текущую ячейку на 1 |
| `>` | Сдвинуть указатель вправо |
| `<` | Сдвинуть указатель влево |
| `[` | Начало цикла — пропустить если ячейка равна 0 |
| `]` | Конец цикла — вернуться если ячейка не равна 0 |
| `.` | Вывести текущую ячейку как символ |
| `,` | Считать символ в текущую ячейку |

## Структура проекта

```
brainfuck-compiler/
├── brainfuck.g4   — формальная грамматика языка в нотации ANTLR4
├── lexer.py       — лексический анализатор
├── parser.py      — синтаксический анализатор, строит AST-дерево
├── ast_nodes.py   — классы узлов AST-дерева
├── printer.py     — печать AST-дерева в консоль
├── semantic.py    — семантический анализатор
├── optimizer.py   — оптимизатор AST-дерева
├── vm.py          — виртуальная машина
└── main.py        — точка входа
```

## Как работает компилятор

Исходный `.bf` файл проходит через несколько этапов:

```
.bf файл
   ↓
Лексер (lexer.py)
Читает файл, выбрасывает всё кроме 8 команд BF.
Возвращает список пар (символ, позиция).
   ↓
Парсер (parser.py)
Строит AST-дерево из токенов.
Обрабатывает вложенные циклы рекурсивно.
Диагностирует синтаксические ошибки (незакрытые скобки и т.д.)
   ↓
Printer (printer.py)
Печатает AST-дерево в консоль с отступами.
   ↓
Семантический анализ (semantic.py)
Проверяет дерево на семантические ошибки:
— пустые циклы []
— слишком глубокая вложенность (> 100 уровней)
— программа ничего не выводит
   ↓
Оптимизатор (optimizer.py)
Упрощает дерево:
— схлопывает одинаковые команды подряд (++ → IncrNode(count=2))
— удаляет взаимоотменяющиеся пары (+- → ничего, >< → ничего)
— удаляет пустые циклы
   ↓
Виртуальная машина (vm.py)
Выполняет оптимизированное дерево.
Лента из 30 000 ячеек, указатель начинается с 0.
```

## Описание файлов

### `brainfuck.g4`
Формальная грамматика языка BrainFuck в нотации ANTLR4. Описывает структуру языка — что является правильной программой. Используется как документация.

### `lexer.py`
Лексический анализатор. Функция `tokenize(source)` принимает строку с исходным кодом и возвращает список пар `(символ, позиция)`. Все символы не являющиеся командами BF выбрасываются.

### `parser.py`
Синтаксический анализатор. Функция `parse(tokens)` принимает список токенов от лексера и строит AST-дерево. Рекурсивно обрабатывает вложенные циклы. При синтаксической ошибке выбрасывает `SyntaxError` с указанием позиции.

### `ast_nodes.py`
Классы узлов AST-дерева: `ProgramNode`, `LoopNode`, `IncrNode`, `DecrNode`, `MoveRNode`, `MoveLNode`, `OutputNode`, `InputNode`.

### `printer.py`
Печатает AST-дерево в консоль с псевдографикой и отступами.

### `semantic.py`
Семантический анализатор. Функция `analyse(tree)` проверяет дерево и возвращает список предупреждений и ошибок.

### `optimizer.py`
Оптимизатор. Функция `optimize(tree)` упрощает AST-дерево для более эффективного выполнения.

### `vm.py`
Виртуальная машина. Класс `VM` с методом `run(tree)` выполняет BF-программу напрямую из AST-дерева.

### `main.py`
Точка входа. Связывает все компоненты по цепочке.
